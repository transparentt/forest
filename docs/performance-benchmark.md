# Forest Performance Benchmark

測定日: 2026-06-21

## 条件

- アプリ: Forest
- ASR: Forest 統合ローカルサーバーの Qwen3-ASR
- カスタマイズ: Ollama `gemma4:e4b`
- カスタマイズ指示: `常に敬語にしてください。`
- 音声: macOS `say` の日本語音声 `Kyoko`
- 測定対象: `/transcribe` と `/customize` の処理時間
- 備考: モデルはウォームアップ後に測定。貼り付け処理や UI 操作時間は含めない。

## 結果

| ケース | 音声長 | ASR | カスタマイズ | 合計 |
|---|---:|---:|---:|---:|
| 超短文 `了解。` | 0.55秒 | 3.71秒 | 1.46秒 | 5.17秒 |
| 短文 `明日の会議は十時からです。` | 1.94秒 | 1.19秒 | 2.26秒 | 3.45秒 |
| タメ口短文 | 2.11秒 | 2.35秒 | 2.03秒 | 4.38秒 |
| 中くらい | 8.36秒 | 3.38秒 | 3.09秒 | 6.47秒 |
| タメ口長め | 11.29秒 | 6.11秒 | 3.43秒 | 9.54秒 |
| 長文 | 22.22秒 | 19.55秒 | 9.19秒 | 28.74秒 |

## サンプル

### タメ口短文

入力:

```text
これやっといて。あとで見とく。
```

書き起こし:

```text
これやっといて、後で見とく。
```

カスタマイズ後:

```text
恐れ入りますが、こちらで対応いたしますので、後ほどご確認いただけますでしょうか。
```

### タメ口長め

入力:

```text
今日の資料ざっと見たけど、まだ直したほうがいいところがあるから、あとでコメント入れとくね。明日の朝までに確認して、必要ならこっちで調整するよ。
```

書き起こし:

```text
今日の資料ざっと見たけどまだ直した方がいいところがあるから後でコメント入れとくね明日の朝までに確認して必要ならこっちで調整するよ。
```

カスタマイズ後:

```text
本日の資料を拝見いたしましたが、いくつか修正が必要な箇所がございますので、後ほどコメントさせていただきます。明日午前中までにご確認いただき、もし必要でしたらこちらで調整いたします。
```

## 所感

- 短文は合計でおおむね 3.5〜5秒。
- 中くらいの文章は 6〜7秒程度。
- 長めのタメ口文章は 10秒前後。
- 22秒程度の長文では 30秒近くかかる。
- 短文では ASR、HTTP、Gemma 呼び出しの固定コストが目立つ。
- 長文では ASR と Gemma の両方が伸びる。
- 1回目の長文カスタマイズでは 40.6秒かかったため、長文カスタマイズは揺れが大きい。

## 改善候補

- 短文向けにカスタマイズを軽量化するプロンプトを用意する。
- 文章がすでに丁寧な場合は Gemma をスキップする判定を入れる。
- 長文はカスタマイズ中 HUD を出し続け、待ち時間が発生していることを明示する。
- ASR とカスタマイズの実測時間をデバッグログに残せるようにする。
